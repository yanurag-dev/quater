# Quater 0.1.0a1

`0.1.0a1` is Quater's first public alpha release.

The release is intentionally narrow: one typed route surface for normal HTTP
clients, MCP tools, and CLI actions, explicit auth boundaries, generated
OpenAPI, generated MCP docs, msgspec JSON handling, and a native router.

## Included

- RSGI-first app object for Granian.
- ASGI and WSGI compatibility adapters.
- Typed route decorators and path/query/body binding.
- Route-level auth hooks.
- MCP `initialize`, `tools/list`, and `tools/call`.
- MCP transport auth with `mcp_auth`.
- Local and remote CLI actions with `cli=True` and `cli_auth`.
- Dry-run and approval hooks for exposed actions.
- `quater dev` and `quater run` server commands backed by Granian.
- OpenAPI JSON, Swagger UI, and human-readable MCP docs.
- msgspec-based serialization.
- Native Rust route matcher.
- CORS, allowed hosts, body limits, secure headers, and signed cookie helpers.

## Still Early

It is an early package for developers who want to try Quater's shared route
model for HTTP, MCP, and CLI operations and give feedback before the public API
becomes harder to change.

The next releases will focus on router grouping, explicit parameter markers,
test-client ergonomics, and security helper APIs.
